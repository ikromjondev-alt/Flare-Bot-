# Discord Модератор-Бот

Бот на **discord.py 2.x** с базой данных **SQLite** — данные (предупреждения, настройки) не
теряются при перезапуске. Плюс отдельная **веб-панель управления** (Flask) и
**мультиязычность** (RU/EN).

## Возможности

- Приветственное сообщение новому участнику + счётчик "N-й участник"
- **`/autorole [channel] [role] [message]`** — одна команда настраивает канал приветствий,
  роль для авто-выдачи и (необязательно) кастомный текст. Включается сразу, без отдельных шагов.
  `/autorole_off` — выключить вручную.
- Модерация — все наказания реальные (через встроенный Discord timeout/бан/кик) и с
  уведомлением нарушителя в личные сообщения о причине:
  - `/mute username time reason` — тайм-аут (максимум 28 дней — ограничение Discord API)
  - `/unmute username reason` — снять тайм-аут
  - `/warn username reason` — предупреждение + тайм-аут на 4 часа
  - `/unwarn username reason` — снять предупреждение и активный тайм-аут
  - `/warnlimit count` — свой порог предупреждений на сервере: при достижении тайм-аут
    продлевается до 1 часа
  - `/ban username days reason` — бан на срок (0 = навсегда)
  - `/unban username reason`
  - `/kick username reason`
- **`/profile [@username]`** — дата регистрации аккаунта, дата захода на сервер, роли, аватар
- **`/support`** — ссылка на ЛС с владельцем бота
- **`/say channel message`** — сообщение от бота в канал (embed)
- **`/dms username message`** — личное сообщение от бота конкретному участнику
- **`/dmall message`** — сообщение в ЛС всем участникам сервера (с обработкой рейт-лимитов и
  отчётом, скольким доставлено)
- **Система жалоб через форум:**
  - Администратор пишет `!setreport` в любом канале (не слэш-команда) — бот присылает
    постоянную панель с категориями нарушений и удаляет исходное сообщение команды
  - Участник выбирает категорию → открывается форма (юзернейм нарушителя + описание) →
    бот просит доказательства (скриншот/видео) в личных сообщениях
  - Бот создаёт (или использует существующий) **Discord Forum Channel** "жалобы" и заводит
    отдельный тред на каждую жалобу с embed: Жалобщик / Нарушитель / Категория / Описание /
    Доказательства / Вердикт администрации
  - Модератор внутри треда жалобы вызывает **`/verdict decision`** — вердикт добавляется
    отдельным полем в исходный embed жалобы
- **Логи** — фиксируют как действия через команды бота, так и **ручные действия модераторов
  напрямую через Discord** (через Audit Log API): вход/выход/кик/бан участника, войс,
  удаление и редактирование сообщений, выдача/снятие ролей и смена ника вручную,
  создание/изменение (с HEX-цветом до/после)/удаление ролей, создание/изменение/удаление
  каналов
- Автоматическое удаление спам-ссылок (с настраиваемым списком разрешённых доменов)
- `/webapp` — приватная команда (видна только владельцу, ID указывается в `.env`) с
  инструкцией, как запустить и открыть веб-панель
- Веб-панель (`webpanel/`) — управление настройками через браузер: каналы, язык, антиспам,
  порог предупреждений, авто-роль, список предупреждений с возможностью снятия

## Установка бота

1. Создайте бота на https://discord.com/developers/applications, во вкладке **Bot**
   включите **Privileged Gateway Intents**: `SERVER MEMBERS INTENT` и `MESSAGE CONTENT INTENT`.
2. Пригласите бота на сервер со следующими правами:
   `Manage Roles, Manage Channels, Kick Members, Ban Members, Moderate Members, Manage Messages, View Channels, Send Messages, Embed Links, Attach Files, Read Message History, View Audit Log, Create Public Threads`
   (Manage Channels и Create Public Threads нужны для авто-создания форума жалоб;
   View Audit Log — для логирования ручных действий модераторов)
3. Установите Python 3.10+ и зависимости:
   ```
   pip install -r requirements.txt
   ```
4. Скопируйте `.env.example` в `.env` и заполните `DISCORD_TOKEN` (и по желанию `GUILD_ID`
   для мгновенной синхронизации команд на одном сервере). Также заполните `OWNER_ID` своим
   Discord ID для команды `/webapp`.
5. Запустите бота:
   ```
   python bot.py
   ```

## Настройка после запуска (на сервере, от имени администратора)

1. `/autorole #канал @роль текст` — авто-роль включится сразу
2. `/logsettings channel #канал-логов`
3. `!setreport` в канале, где должна появиться панель подачи жалоб (форум "жалобы" создастся
   автоматически при первой жалобе)
4. `/antispam on` (включён по умолчанию) и при необходимости `/antispam allow домен.com`
5. `/warnlimit 3` (или любое число) — авто-эскалация тайм-аута при накоплении предупреждений.
   `/warnlimit 0` — отключить.
6. `/language` — выбрать русский или английский язык для сервера

## Веб-панель управления

Панель — отдельное Flask-приложение, читающее ту же базу `bot_database.db`. Запускать нужно
на той же машине (или с доступом к тому же файлу БД), что и бот.

1. В `.env` заполните `WEBPANEL_PASSWORD` и `WEBPANEL_SECRET_KEY`.
2. Запустите:
   ```
   cd webpanel
   python app.py
   ```
3. Откройте `http://localhost:5000` (или ваш адрес/порт), войдите по паролю.

## Структура проекта

```
bot.py                    — точка входа бота
config.py                 — загрузка настроек из .env
database.py                — работа с SQLite
i18n.py                     — словари переводов RU/EN
cogs/
  welcome.py                 — приветствие новых участников
  autorole.py                — /autorole, /autorole_off
  moderation.py               — mute/unmute/warn/unwarn/ban/unban/kick/profile/say/dms/dmall
  report.py                    — !setreport, форум жалоб, /verdict
  support.py                    — /support
  help_cmd.py                    — /help
  logging_cog.py                  — логи команд бота + ручных действий модераторов
  antispam.py                      — фильтр спам-ссылок
  language.py                       — /language
  webapp.py                          — /webapp (только владелец)
webpanel/
  app.py                              — Flask-приложение веб-панели
  db.py                                — синхронный доступ к той же SQLite базе
  templates/                           — HTML-шаблоны панели
```

## Примечания

- `/mute` и `/warn` используют встроенный **Discord timeout**, а не отдельную роль — Discord
  ограничивает тайм-аут максимум **28 днями** (лимит самого API).
- После любого наказания (`/mute`, `/warn`, `/ban`, `/kick`) бот пытается написать нарушителю
  в личные сообщения с причиной. Если у человека закрыты ЛС — это тихо игнорируется, само
  наказание при этом всё равно применяется.
- Форум для жалоб создаётся ботом автоматически при первой жалобе (если бота не лишили прав
  `Manage Channels`/`Create Public Threads`). Если создать не удалось — используется запасной
  текстовый канал из `/reportsettings channel`.
- Логи ручных действий модераторов через Discord (не через бота) требуют права `View Audit
  Log` и могут иметь небольшую задержку — это ограничение самого Discord API, не бота.
- Веб-панель и бот — два независимых процесса; запускайте их отдельно (`python bot.py` и
  `python webpanel/app.py`).
